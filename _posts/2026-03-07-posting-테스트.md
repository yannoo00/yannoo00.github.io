### 블로그를 벗어나지 않고 포스팅 하기

그동안은 깃허브 블로그에 포스팅 하기 위해서 vscode를 켜고 .md 파일을 생성하여 글을 작성했다.  

md파일을 생성하고 글 작성 이후에는 commit&push를 해줘야 올라가기 때문에, 
글 하나 쓰려고 해도 코드 편집기를 열어서 이것 저것 해줄게 많아 귀찮았다.  

편하게 포스팅 할 수 있는 방법을 찾다가 notion에 github API 붙이기, 혹은 다양한 github API를 활용해서 직접 편집기를 블로그 내에 구현하기 등의 방법을 찾았고  
이참에 claude code를 활용하여 더 적극적으로 블로그를 편집해보기로 했다.   

<img width="1367" height="815" alt="Image" src="https://github.com/user-attachments/assets/d4b93bf7-820d-41ab-a188-3d87b3d692db" />   

우선 글 편집 및 새 글 작성까지 구현해둔 상태이다.  
github action을 통해 배포되기에 정적 사이트의 배포만 가능한데, Github API를 활용하면 웬만한 블로그 기능은 모두 간단하게 구현할 수 있다.  


#### 인증 
/admin으로 접근하여 토큰을 인증하면 아래 기능에 접근할 수 있다.  
한 번 인증하면 localStorage에 토큰을 저장한다.  



#### 텍스트 에디터  

[Easy MD Editor](https://github.com/Ionaru/easy-markdown-editor) 라이브러리를 사용했다.  
md파일 작성을 편리하게 도와주는 라이브러리이고, 
```javascript

  function initMDE(body) {
    _mde = new EasyMDE({
      element: document.getElementById('fab-md-textarea'),
      spellChecker: false,
      autosave: { enabled: false },
      toolbar: [
        'bold',
        'italic',
        'heading',
        '|',
        'code',
        'quote',
        '|',
        'unordered-list',
        'ordered-list',
        '|',
        'link',
        'image',
        '|',
        'preview',
        'side-by-side',
        'fullscreen',
      ],
      minHeight: '500px',
    });
    _mde.value(body || '');
    document.getElementById('fab-save-btn').addEventListener('click', save);
    document.getElementById('fab-cancel-btn').addEventListener('click', cancel);
  }
```
이처럼 간단하게 불러와 사용할 수 있다. 편집 버튼으로 접근하는 경우 인자로 넘기는 body에 기존 게시글의 텍스트가 들어오게 된다.  
세심한 opus가 이 라이브러리의 cdn을 편집/새 글 버튼을 눌렀을 때 load하게 하여 일반 독자의 메모리 낭비를 줄여줬다.   

```javascript
async function save() {
    var title = document.getElementById('fab-fm-title').value.trim();
    var date = document.getElementById('fab-fm-date').value.trim();
    var cats = document.getElementById('fab-fm-categories').value.trim();
    var tags = document.getElementById('fab-fm-tags').value.trim();
    var excerpt = document.getElementById('fab-fm-excerpt').value.trim();
    var body = _mde.value();

    if (!title) {
      showNotice('제목을 입력하세요.', 'error');
      return;
    }
    if (!date) {
      showNotice('날짜를 입력하세요.', 'error');
      return;
    }

    var content =
      buildFrontMatter(title, date, cats, tags, excerpt) + '\n' + body;
    var encoded = btoa(
      String.fromCharCode.apply(null, new TextEncoder().encode(content)),
    );

    var isNew = !_file;
    var path = isNew
      ? POSTS_PATH + '/' + date + '-' + slugify(title) + '.md'
      : _file.path;
    var payload = {
      message: (isNew ? 'Create: ' : 'Update: ') + title,
      content: encoded,
      branch: BRANCH,
    };
    if (!isNew) payload.sha = _file.sha;

    var saveBtn = document.getElementById('fab-save-btn');
    saveBtn.disabled = true;
    saveBtn.textContent = isNew ? '발행 중...' : '저장 중...';

    try {
      var result = await ghFetch(
        '/repos/' + OWNER + '/' + REPO + '/contents/' + path,
        {
          method: 'PUT',
          body: JSON.stringify(payload),
        },
      );
      if (isNew) {
        _file = { path: result.content.path, sha: result.content.sha };
        saveBtn.textContent = '저장';
        showNotice(
          '발행되었습니다! GitHub Actions가 빌드를 시작합니다.',
          'success',
        );
      } else {
        _file.sha = result.content.sha;
        saveBtn.textContent = '저장';
        showNotice(
          '저장되었습니다! GitHub Actions가 빌드를 시작합니다.',
          'success',
        );
      }

```
payload에 sha를 포함해서 보내면 이미 존재하는 게시글을 수정하게 된다.  
sha 값이 하는 역할은 서버의 sha와 현재 post의 sha를 비교하여 파일이 수정되었는지를 체크하는 것이다.  
(내가 혼자 쓰는 이 경우에는 쓸모 없지만 동시성을 고려한 방어책이다)  

```  '/repos/' + OWNER + '/' + REPO + '/contents/' + path,``` 
 url로 수정/추가 대상인 파일을 지정하고 payload의 message가 커밋 메시지, branch가 푸시 대상 브랜치가 된다.
