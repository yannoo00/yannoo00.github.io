property는 클래스 내부 필드에 대한 getter, setter의 구현을 간단하게 돕는다.  
함수이지만 읽고 쓸 때 마치 변수처럼 작성하여 사용할 수 있다.  

```cs
private int hp;
public int Hp
{
    get { return hp; }
    set 
    {
        if (value < 0) hp = 0; // 체력이 음수가 되는 것을 방지
        else hp = value;
    }
}
```  
위처럼 getter와 setter를 정의할 때 함수처럼 내부에 제약 등 구체적인 사항을 구현할 수 있다.    

```cs
    public class LobbyPacketHandler
    {
        // PacketId -> 핸들러 함수 테이블
        // LobbySession.OnRecvPacket()에서 이 테이블을 조회해 호출
        public static Action<PacketSession, ArraySegment<byte>>[] Handlers { get; private set; }
```  
packet Handler들을 담은 델리게이터를 필드가 아닌 프로퍼티로 만들고,  
get은 누구나 할 수 있게 public으로 열었지만 set은 Handlers = new ... 처럼 덮어써버릴 수 없게 막았다.
