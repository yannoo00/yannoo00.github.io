네트워크 계층과 각 계층에 대표되는 주소 체계에 대해 거시적으로 정리한다.  
보통 전공 수업 등에서는 OSI 7 layer로 처음 접하지만, 더 실용적으로는 4계층으로 이해하고 활용게 되는 것 같다.  

4계층은 application - transport - internet - link 로 구분한다.  각 계층에서 활용하는 주소체계를 기반으로 계층의 역할을 가늠해보자.  

application 계층의 경우, 사용자(=우리)가 활용하는 주소를 그대로 생각하면 된다. (www.naver.com 같은 도메인 이름) 주소를 입력하고 -> DNS 서버를 거쳐 받아온 IP주소를 기반으로 Transport(포트) - Internet(IP) - Link(MAC) 계층으로 데이터가 차례대로 캡슐화되며 실제 통신이 시작된다.


#### Transport layer - port 

네트워크는 정보의 전달을 위한 개념이다.  
정보 전달의 이해를 위해 end to end로 정보가 전달된다고 가정하고, 이 때의 'end'란 무엇인가?  
application 계층에서의 데이터 전송을 고려하면 최종 목적지는 process가 된다.  
어떤 PC의 OS 위에서 실행되는 프로세스를 식별하기 위해 **port 번호**를 활용한다.  
로컬에서 웹 프로젝트 등을 빌드하여 127.0.0.1:8080과 같은 주소에서 실행해본 경험이 있을 수 있다.  
이 때의 8080이 포트번호가 된다.  


#### Internet layer - IP
그 앞의 127.0.0.1은 흔히 인터넷 프로토콜, **IP주소**라고 말하는데 127.0.0.1은 IPv4 번호이고, ([IPv6도 사용하지만](https://namu.wiki/w/IP%20%EC%A3%BC%EC%86%8C) 일단 패스.)  
특정 물리적 제품에 1:1로 대응되는 개념이 아니다. 스마트폰으로 공유기에 연결하여 통신하는 경우를 생각해보자.  
<img width="566" height="282" alt="Image" src="https://github.com/user-attachments/assets/0a551b0e-9bfc-4c45-a0b1-ca164191a2ba" />  

공유기(Router)는 두 개의 서로 다른 네트워크 망을 연결한다.  
WAN(외부망)에 대해서는 인터넷 서비스 제공자(ISP)와 연결되어, 전 세계에서 식별 가능한 유니크한 공인 IP를 할당받는다.  
LAN(내부망)에 대해서는 내부적으로 통신하기 위해 라우팅되지 않는 사설 IP 대역을 생성하고, 자신의 IP를 설정한다.  


이 라우터에 기기(스마트폰 등)가 연결될 때, 라우터 내부의 **Dynamic Host Configuration Protocol(DHCP)**가 작동하여 기기가 통신하는데 필요한 네트워크 정보를 구성해준다. 이 때 사설 IP 주소 등이 설정된다.  

할당받은 사설 IP 주소는 내부망에서만 유효하므로, 라우터를 통해 외부 인터넷으로 패킷이 나갈 때에는 패킷의 IP 헤더를 조작하는 **Network Address Translation(NAT) 과정**을 수행한다. 이를 통해 라우터 자신의 공인 IP에, 이 라우터를 통한 특정 기기를 식별하기 위해 포트 번호를 수정하여 요청을 보내고(다른 기기이지만 나갈 때는 같은 공인 IP로 나가게 되므로) 다시 요청(패킷)이 들어왔을 때 이 포트 번호를 활용하여 기기에 전달한다.  

폰-공유기의 관계가 아니라 랜선에 데스크탑을 꽂아 사용하는 상황은 어떨까?  
이는 랜선에 직접 꽂았느냐 보다는 라우터를 거치냐, 그렇지 않느냐에 따라 달라진다.  
사설 IP가 사라지고 공인 IP를 직접 할당받게 된다. NAT 과정도 생략된다.  


#### Link layer - MAC

<img width="650" height="372" alt="Image" src="https://github.com/user-attachments/assets/1ae32d24-ab1b-4727-9429-bf8c5eaa1921" />  

스위치는 유선 연결 기반으로, MAC 주소를 읽어 내부망의 연결을 담당한다.  
사진의 **Router와 switch를 하나로 합친 기기가 흔히 사용하는 공유기**다.  
router는 외부 연결을 담당하고, 스위치는 연결된 기기들 사이의 내부망 연결을 담당한다.  
내부망 연결은 IP로 하는 것이 아니라 기기의 **Network Interface Card(NIC)** 에 새겨진 MAC 주소를 기반으로 이뤄진다.
